## 📖 Overview

<a href="#appleCompressor">Apple Compressor</a> serves Final Cut Pro editors who need delivery formats beyond what Final Cut Pro's built-in sharing provides, independent producers delivering to broadcast networks and streaming platforms, post-production facilities creating deliverables to client specifications, and anyone in the Apple-platform production ecosystem who needs professional encoding control.

The <a href="#appleCompressor">Final Cut Pro editor delivery workflow</a> is Compressor's core use case — an editor finishing in Final Cut Pro who needs to deliver to a broadcast specification, a streaming platform's technical requirements, or a client's custom codec settings. Compressor extends Final Cut Pro's delivery capability from consumer-oriented sharing to professional deliverable creation. The <a href="#appleCompressor">broadcast and streaming platform delivery use case</a> prepares files to the specific technical specifications that networks, streaming services, and distribution platforms publish — resolution, codec, bitrate, audio format, subtitle format, and metadata requirements that vary by platform and require precise encoding parameter control.

The <a href="#appleCompressor">iTunes Store and Apple original content submission</a> use case applies Compressor's Apple-specific submission tools for content delivered to Apple's platforms — the submission package format, audio configuration, and metadata requirements that iTunes Store and Apple TV+ content delivery requires. <a href="#appleCompressor">One-time purchase from Mac App Store</a> — Compressor is available as a standalone one-time purchase alongside its availability as the Final Cut Pro companion encoding application.

<a href="#appleCompressor">GPU-accelerated encoding</a> uses the Mac's GPU for hardware-accelerated H.264 and HEVC encoding — significantly faster than software encoding for large file exports, making high-resolution encoding practical within production timelines. <a href="#appleCompressor">Subtitle and caption support</a> embeds subtitle files and closed captions into output files — the accessibility and localization features that broadcast delivery and streaming platform submission requires.

<a href="#appleCompressor">Custom destinations</a> configure recurring delivery specifications as saved destinations — creating named presets for each client, platform, or delivery requirement that apply to any source with a single selection. <a href="#appleCompressor">Audio format flexibility</a> encodes audio in stereo, 5.1, and other multichannel configurations, with Dolby Atmos packaging for streaming platforms that support spatial audio delivery.

## ❓ FAQ

<details>
<summary>How does Compressor compare to HandBrake?</summary>

Compressor provides professional delivery format support, Final Cut Pro integration, iTunes Store packaging, distributed processing, and Apple-platform specific features. HandBrake is a free open-source encoder focused on H.264/HEVC compression for general use. Compressor serves professional post-production delivery requirements that HandBrake doesn't address.

</details>

<details>
<summary>Does Compressor run on Apple Silicon?</summary>

Yes. Compressor is optimized for Apple Silicon with hardware acceleration using the M-series chip's media encoding capabilities.

</details>

Yes. Subtitle files and closed captions embed into output files for broadcast delivery and streaming platform accessibility requirements.
